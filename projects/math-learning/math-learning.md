# AI/CV 学生需要学多少数学？

## 一句话结论

**不需要数学家级别的证明能力，但核心三个方向必须"能读懂 + 能直觉理解"。**

数学之于 AI 学生，就像语法之于写作——不会语法写不出好文章，但不需要成为语言学教授。尤其是跨考选手，数学是拉开差距的利器。

## 核心心态

> 数学不是用来背的——是用来"看穿"的。你看穿了一个公式，它就是你的武器。

1. **不求证明，求直觉**：能说清楚"这个公式在干什么"比能证明它重要 100 倍
2. **代码即数学**：`torch.matmul(X, W)` 比 \(\mathbf{X}\mathbf{W}\) 更实在，两边对照着学
3. **三维重建是你的秘密武器**：你是规划出身，对空间的理解力天然比纯 CS 学生强。射影几何、多视图几何对你来说反而有"体感"——别怕公式，代码跑出来你就明白了
4. **矩阵求导不用死记**：Matrix Cookbook + 链式法则 = 一切

> 跨考不可怕，数学一 76 分不是负担是底气。CS 科班生学数学的时候你在画总规图——现在你在学 CV，思路反而比他们更立体。🔥

## 📖 概念速查

> 学一个记一个，慢慢攒。能跟代码对照最好。

### 矩阵乘法（Matrix Multiplication）

神经网络前向传播的本质：

```python
# 输入 X (batch_size × input_dim) × 权重 W (input_dim × output_dim)
# → 输出 (batch_size × output_dim)
output = X @ W  # 或 torch.matmul(X, W)
```

一个全连接层就是一次矩阵乘法 + 一次加法（bias）。

### 梯度（Gradient）

"往哪个方向走 loss 下降最快"的数学描述：

$$\nabla_\theta L = \frac{\partial L}{\partial \theta}$$

```python
loss.backward()  # PyTorch 自动帮你算梯度
optimizer.step() # 沿着梯度方向更新参数
```

> 跟你在 Grasshopper 里跑梯度下降做平面布局优化是一样的——只是从 2D 平面换成了百万维参数空间。

### SVD（奇异值分解）

三维重建里最常用的矩阵分解：

$$A = U \Sigma V^T$$

```python
U, S, Vt = np.linalg.svd(A)
# 本质矩阵 E 的秩 2 约束就是靠 SVD 来实现的
```

> 做 Structure from Motion 的时候，基本每一步都在跟 SVD 打交道。

### 贝叶斯定理（Bayes' Theorem）

$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$

"在看到数据 B 之后，我对 A 的信念怎么更新"——这是所有概率模型的底层逻辑。分类器输出的 softmax 值本质上就是 \(P(\text{class} | \text{image})\)。

### 交叉熵（Cross-Entropy）

分类任务最常用的 Loss：

$$H(p, q) = -\sum_x p(x) \log q(x)$$

```python
loss = F.cross_entropy(logits, labels)  # PyTorch 一行搞定
```

> 直觉：预测分布 q 跟真实分布 p 差多远。越远 loss 越大，SGD 会逼着 q 靠近 p。

### 射影几何（Projective Geometry）🔵

三维重建的数学基础。核心思想：**用高一维的坐标表示低维空间**（齐次坐标）。

- 2D 点 \((x, y)\) → 齐次 \((x, y, 1)\)
- 3D 点 \((X, Y, Z)\) → 齐次 \((X, Y, Z, 1)\)
- 相机投影：\(\mathbf{x} = K[R|t]\mathbf{X}\)（一个 3×4 矩阵把 3D 世界投到 2D 图像）

> 这是你 CV + 三维重建方向的独家优势所在——规划本科的空间思维直接复用。
